# minuet-ai.nvim Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add minuet-ai.nvim to the Neovim config using local Ollama/qwen2.5-coder for AI completions surfaced both as a blink.cmp menu source and as inline virtual text.

**Architecture:** One new plugin file (`lua/plugins/minuet-ai.lua`) handles all minuet setup; `blink-cmp.lua` gains a `providers.minuet` entry + `"minuet"` in the sources list; `which-key.lua` gains two `<leader>a` keymaps; `install.sh` and `init.lua` get one-line additions each.

**Tech Stack:** minuet-ai.nvim, blink.cmp (existing), Ollama (local), qwen2.5-coder model

### Task 2: Create lua/plugins/minuet-ai.lua

**Files:**
- Create: `lua/plugins/minuet-ai.lua`

- [ ] **Step 1: Create the plugin file**

Create `lua/plugins/minuet-ai.lua` with the following content:

```lua
-- =============================================================================
-- lua/plugins/minuet-ai.lua
-- minuet-ai.nvim — AI code completion via local Ollama
--
-- Provider: Ollama (local, no API key required)
-- Model:    qwen2.5-coder:latest
--
-- Completions surface in two ways:
--   1. blink.cmp menu — appears alongside LSP/snippet items (Tab/CR to accept)
--   2. Virtual text   — inline ghost text (<A-y> accept, <A-e> dismiss)
--
-- Prerequisites:
--   ollama pull qwen2.5-coder
--   ollama serve   (or ensure the launchd/systemd service is running)
--
-- To install (run once in your shell):
--   git clone --depth 1 https://github.com/milanglacier/minuet-ai.nvim \
--     ~/.config/nvim/pack/local/opt/minuet-ai.nvim
-- =============================================================================

vim.cmd.packadd("minuet-ai.nvim")

require("minuet").setup({

  -- ── Provider ──────────────────────────────────────────────────────────────
  provider = "ollama",

  -- ── Timing ────────────────────────────────────────────────────────────────
  -- Debounce completions so Ollama isn't hit on every keystroke.
  throttle         = 1500,   -- ms between requests
  request_timeout  = 3,      -- seconds before giving up

  -- ── Provider options ──────────────────────────────────────────────────────
  provider_options = {
    ollama = {
      model      = "qwen2.5-coder:latest",
      stream     = true,
      end_point  = "http://localhost:11434/api/generate",
      optional   = {
        num_predict = 256,   -- max tokens per completion
      },
    },
  },

  -- ── Virtual text ──────────────────────────────────────────────────────────
  -- Ghost text rendered inline after the cursor.
  -- Accepted with <A-y>; dismissed with <A-e>.
  virtualtext = {
    -- Enable for all filetypes. Remove specific ft strings to restrict.
    auto_trigger_ft = { "*" },
    keymap = {
      accept      = "<A-y>",   -- accept full suggestion
      accept_line = "<A-l>",   -- accept one line at a time
      dismiss     = "<A-e>",   -- dismiss suggestion
      prev        = "<A-[>",   -- cycle to previous suggestion
      next        = "<A-]>",   -- cycle to next suggestion
    },
  },
})
```

- [ ] **Step 2: Verify Neovim loads without errors**

### Task 3: Wire minuet into blink.cmp

**Files:**
- Modify: `lua/plugins/blink-cmp.lua`

This adds minuet as a named blink provider and inserts `"minuet"` into the active sources lists. The `score_offset = -3` keeps LSP items ranked above AI suggestions.

- [ ] **Step 1: Add the providers table to blink-cmp.lua**

In `lua/plugins/blink-cmp.lua`, find the `sources = {` block (currently ends after `per_filetype`). Add a `providers` key so the full `sources` table reads:

```lua
  sources = {
    default = { "lsp", "path", "snippets", "buffer", "minuet" },

    per_filetype = {
      markdown = { "obsidian", "obsidian_tags", "obsidian_new", "lsp", "path", "snippets", "buffer", "minuet" },
    },

    providers = {
      minuet = {
        name   = "minuet",
        module = "minuet.blink",
        score_offset = -3,
      },
    },
  },
```

- [ ] **Step 2: Verify Neovim loads without errors**

```bash
nvim --headless -c "lua require('plugins.blink-cmp')" -c "lua require('plugins.minuet-ai')" -c "qa" 2>&1
```
Expected: silent exit.

- [ ] **Step 3: Smoke-test completion in a live Neovim**

Open any file in Neovim and trigger completion (`<Tab>` in insert mode). Confirm the menu appears and LSP items are still present. With Ollama running and warmed up you should also see a "minuet" kind item appear after a short delay.

- [ ] **Step 4: Commit**

```bash
git add lua/plugins/blink-cmp.lua
git commit -m "feat: add minuet as blink.cmp source"
```

### Task 4: Add which-key entries

**Files:**
- Modify: `lua/plugins/which-key.lua`

- [ ] **Step 1: Confirm toggle function name**

From the grep you ran in Task 1 Step 4, identify the correct toggle function. Common possibilities:
- `require("minuet.virtualtext").toggle_auto_trigger()`
- `require("minuet").toggle_auto_trigger()`

Run this in Neovim to verify:
```vim
:lua require("minuet.virtualtext").toggle_auto_trigger()
```
If it errors, try `require("minuet").toggle_auto_trigger()`. Use whichever works.

- [ ] **Step 2: Add the two keymaps to which-key.lua**

In `lua/plugins/which-key.lua`, find the `<leader>a` group line:
```lua
  { "<leader>a",  group = "ai / claude" },
```

In the `wk.add({` block, find the section with Claude Code keymaps (lines with `<leader>ac`, `<leader>aR`, `<leader>aV`) and append after them:

```lua
  -- ── Minuet AI ─────────────────────────────────────────────────────────────
  {
    "<leader>am",
    function() require("minuet.virtualtext").toggle_auto_trigger() end,
    desc = "Minuet: toggle auto-suggest",
  },
  {
    "<leader>aM",
    function() require("minuet").complete() end,
    desc = "Minuet: complete (manual)",
  },
```

> If Task 1 Step 4 revealed the toggle lives elsewhere, substitute the correct module path.

- [ ] **Step 3: Verify which-key popup shows the new entries**

Open Neovim, press `<leader>a` and wait for the popup. Confirm `am` and `aM` appear with their descriptions.

- [ ] **Step 4: Commit**

```bash

git commit -m "feat: add minuet which-key entries under <leader>a"
```
